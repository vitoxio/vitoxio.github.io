Typing is Hard with Vittorio Corbo

What is this? Typing is Hard I first had the idea of creating a typing game when I first got into college. I wanted another way to study through my notes, as I found that reading them out wasn't as engaging. So combining this, with a new found interest in how to improve my typing, by using websites such as keybr.com, I decided to create a simple piece of python code that enabled me to type my notes out.

This was all this idea was for quite some time. A simple code that would take a text file to type out. Until some time later I had to create an original game for a game studies class. Inspired in the readings on player agency and gamification, I decided to expand on this simple code, and turn it into a game.

But you might be asking: what makes this typing game unique? Well its simple, it's the most bearbones and least "vidoegamey" typing game possible. At first this sounds like a huge boon for my game, until one realises how bad the average typing games is. As these games are overly gamified. The average typing game implores the user to type increasingly fast as enemies come onto the screen until they reach can't keep up anymore and die. However, I feel that typing is similar to how one learns how to read. One does not get better at reading by trying to speedrun a book, this is not how kids transition into reading childrens books to Harry Potter. This growth comes from routine practice. Which slowly as time goes on, as one reads more and more, one gets more familiar and comfortable reading, thus becoming a better reader. So it is this principle I used at the basis of my typing game.

In the game, players are only required to type what is on screen, there is no music, no feedback for wrong keystrokes, nothing. So, it creats this safe environment where the worst of typers can practice without any negative feedback to deter their growth. So, the game is destilled to just the essence of typing. It's still a game, as there is a tug of war between what the player needs to type, vs the game, which makes the player need to write ridiculous lines to progress. The best thing is once the game is beaten, options are added to costumise the typing experience, so players can add errors made while typing, words per minute, etc. Allowing the game to be costumized to ones personal typing goals. Which is then further cemented by the ability to change the games .txt files, to whatever custom text the user wants to use. Which allows me to be able to still study my class notes in peace. (Made with GameMakerStudio 2)

ALPHABET

What is this? ALPHABETThis game was made for the GMTK 2020 Game Jam. With the theme of: Out of Control. I feel that the best mindset of any Game Jam is to keep it simple. Feature creep is something that easily seeps into game production. So to keep things in check I always try to create only the most essential aspects of a game first, get them working, and only after, if there is time, add other features. So for this Jam, I decided to make a snake-like esque game in GameMakerStudio 2. The catch of the game is that when grabbing a letter, it changes the players keybindings into the letter they grabbed.

The idea of the game is to work similar to the family game Twister. Where the player must contort into strange positions to win. So ALPHABET messes with the players muscle memories, as they must fight their instincts to move around. This is then further elevated by the games arcady nature. As it incites players to get better times, making them replay the game to get better and fight their muscle memory.

Two things I feel that sneak under the radar when playing the game, has to do with the games namesake: ALPHABET. When grabbing letters the player will notice that notes start to play. Most players will not understand why this music is playing, mainly due to how slow most players are. However, the music that is playing is Twinkle-Twinkle little star, which is coincidentally the alphabet song. Hence, the games name.

As an aside, I am really proud of my performance on the Game Jam, as I came out 434 out of 5477 game submissions, which is especially good considering this was my first serious Game Jam, and moreover the largest Game Jam ever made at the time. As a whole, I hold this game close, not only because of how it did, but rather by giving an unique spin into the snake-like game genre.

EVERY (few) WORDS ARE RANDOM

What is this? EVERY (few) WORDS ARE RANDOM was made for GMTK's 2022 game jam, whose theme was: Roll of the Dice. From there I thought back to MadLibs, and how people would choose the most random words to fill out sentences. Hence I thought, what if the choice of words was 100% random? This is that game
